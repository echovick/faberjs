# Encryption & Hashing

`@faber-js/crypt` provides two distinct concerns that are often conflated but serve different purposes:

- **Hashing** — one-way transformation for passwords. You can never recover the original value; you only verify that a candidate matches.
- **Encryption** — two-way transformation for sensitive data you need to retrieve (API keys, OAuth tokens). Requires `APP_KEY`.
- **Signed URLs** — tamper-proof URLs with an optional expiry, used for email verification, password reset links, and temporary file downloads.

Similar to Laravel's `Hash`, `Crypt`, and `URL` facades, FaberJS exposes dedicated objects for each concern.

---

## Installation

```bash
pnpm add @faber-js/crypt bcryptjs
pnpm add -D @types/bcryptjs
```

## Configuration

### Register the provider

```typescript
// bootstrap/app.ts
import { CryptServiceProvider } from '@faber-js/crypt';
import cryptConfig from '../config/crypt';

app.register(new CryptServiceProvider(app, cryptConfig));
```

### Create the config file

```typescript
// config/crypt.ts
import { env } from '@faber-js/config';

export default {
  // Generated by: npx faber key:generate
  key: env('APP_KEY', ''),

  bcrypt: {
    // Cost factor — higher = slower = more secure; 12 is the recommended minimum
    rounds: Number(env('BCRYPT_ROUNDS', '12')),
  },
};
```

### `.env` reference

```ini
APP_KEY=base64:AbCdEfGhIjKlMnOpQrStUvWxYz0123456789AAAA=
BCRYPT_ROUNDS=12
```

### Generating an application key

```bash
npx faber key:generate
```

This generates a cryptographically random 256-bit key and writes it to `.env` as a base64-encoded string. **Never share or commit your `APP_KEY`.** If it is compromised, all encrypted values stored in your database must be re-encrypted with a new key.

::: warning Key rotation
Changing `APP_KEY` invalidates all previously encrypted values. Rotate the key only when you are prepared to re-encrypt existing data.
:::

## Hashing

### `Hash.make`

Hash a plain-text password before storing it:

```typescript
import { Hash } from '@faber-js/crypt';

const hashed = await Hash.make('secret-password');
// '$2b$12$...'
```

### `Hash.check`

Verify a password against a stored hash:

```typescript
const isValid = await Hash.check('secret-password', user.getAttribute('password') as string);

if (!isValid) {
  throw new UnauthorizedException('Invalid credentials.');
}
```

### `Hash.needsRehash`

Check whether a stored hash was created with a weaker cost factor than the current `BCRYPT_ROUNDS` setting. Use this after a successful login to transparently upgrade old hashes:

```typescript
const stored = user.getAttribute('password') as string;

if (await Hash.needsRehash(stored)) {
  const upgraded = await Hash.make(plainText);
  await user.update({ password: upgraded });
}
```

### Full login example

```typescript
// app/controllers/AuthController.ts
import { Injectable } from '@faber-js/core';
import { Controller } from '@faber-js/router';
import { Hash } from '@faber-js/crypt';
import { Auth } from '@faber-js/auth';
import type { Request } from '@faber-js/http';
import { Response } from '@faber-js/http';
import { User } from '../models/User';

@Injectable()
export class AuthController extends Controller {
  async register(req: Request): Promise<Response> {
    const { name, email, password } = req.validated() as Record<string, string>;

    const user = await User.create<User>({
      name,
      email,
      password: await Hash.make(password),
    });

    return this.json({ data: user }, 201);
  }
}
```

## Encryption

Encryption is suitable for values you must be able to retrieve — OAuth access tokens, API credentials, sensitive user data. The payload is AES-256-GCM encrypted and HMAC-signed so you can detect tampering.

### `Crypt.encryptString`

```typescript
import { Crypt } from '@faber-js/crypt';

const encrypted = await Crypt.encryptString('my-oauth-token');
// 'eyJpdiI6...' — a base64-encoded JSON envelope
```

Store this string in your database.

### `Crypt.decryptString`

```typescript
const token = await Crypt.decryptString(encrypted);
// 'my-oauth-token'
```

### `DecryptException`

Decryption throws `DecryptException` when the ciphertext has been tampered with, the key is wrong, or the payload is malformed:

```typescript
import { Crypt, DecryptException } from '@faber-js/crypt';

try {
  const value = await Crypt.decryptString(stored);
  return this.json({ token: value });
} catch (err) {
  if (err instanceof DecryptException) {
    throw new HttpException('Could not decrypt the stored value.', 500);
  }
  throw err;
}
```

### Storing encrypted model attributes

```typescript
// Store
await user.update({
  github_token: await Crypt.encryptString(githubAccessToken),
});

// Retrieve
const token = await Crypt.decryptString(user.getAttribute('github_token') as string);
```

::: tip Casts coming in v2.2
`@faber-js/orm` v2.2 will add `encrypted` model casts so the encrypt/decrypt cycle happens transparently in the model. For now, call `Crypt` explicitly in your service layer.
:::

## Signed URLs

Signed URLs are ordinary URLs with an HMAC signature appended as a query parameter. The `signed` middleware verifies the signature on every incoming request to protect resources that should only be accessible via a valid, unmodified link.

Common use cases: email verification, password reset, one-click unsubscribe, temporary file downloads.

### `URL.signedRoute`

Generate a URL that is valid indefinitely (no expiry):

```typescript
import { URL } from '@faber-js/crypt';

const verifyUrl = URL.signedRoute('email.verify', {
  id: user.getAttribute('id'),
  hash: user.getAttribute('email_hash'),
});
// https://app.example.com/email/verify?id=42&hash=abc123&signature=...
```

### `URL.temporarySignedRoute`

Generate a URL that expires after a given number of seconds:

```typescript
// Expires in 1 hour (3600 seconds)
const resetUrl = URL.temporarySignedRoute('password.reset', 3600, {
  email: user.getAttribute('email'),
});
// https://app.example.com/password/reset?email=alice%40example.com&expires=1745000000&signature=...
```

### Routes and the `signed` middleware

Protect the route by registering the `SignedMiddleware` and applying it:

```typescript
// bootstrap/app.ts — register the middleware
import { SignedMiddleware } from '@faber-js/crypt';

kernel.register('signed', new SignedMiddleware());
```

```typescript
// routes/api.ts
Route.get('/email/verify', [EmailController, 'verify']).middleware(['signed']);
Route.get('/password/reset', [PasswordController, 'reset']).middleware(['signed']);
```

When the signature is invalid or the link has expired, `SignedMiddleware` returns a `403 Forbidden` response automatically.

### Full password reset flow

```typescript
// 1. Request a reset — generates a signed URL and emails it
@Injectable()
export class PasswordController extends Controller {
  async sendResetLink(req: Request): Promise<Response> {
    const email = req.input('email') as string;
    const user = await User.where<User>('email', email).first();

    if (user) {
      const resetUrl = URL.temporarySignedRoute('password.reset', 3600, { email });
      await Mail.to(email).send(new PasswordResetMail(email, resetUrl));
    }

    // Always return 200 to avoid email enumeration
    return this.json({ message: 'If that address exists, a reset link has been sent.' });
  }

  // 2. Handle the reset — middleware has already validated the signature
  async reset(req: Request): Promise<Response> {
    const { email, password } = req.validated() as Record<string, string>;

    const user = await User.where<User>('email', email).firstOrFail<User>();
    await user.update({ password: await Hash.make(password) });

    return this.json({ message: 'Password reset successfully.' });
  }
}
```

## Testing

### Hashing

During tests, hashing with bcrypt is slow. Override `BCRYPT_ROUNDS` to `4` in your test environment:

```ini
# .env.test
BCRYPT_ROUNDS=4
```

### Crypt fake

`Crypt.fake()` replaces encrypt/decrypt with identity functions (base64 encode/decode) so tests run without a real `APP_KEY`:

```typescript
import { Crypt } from '@faber-js/crypt';

beforeAll(() => {
  Crypt.fake();
});

test('stores encrypted token', async () => {
  const encrypted = await Crypt.encryptString('token-value');
  const decrypted = await Crypt.decryptString(encrypted);

  expect(decrypted).toBe('token-value');
});
```

### Signed URL testing

```typescript
import { URL } from '@faber-js/crypt';

test('signed route is valid', () => {
  const url = URL.signedRoute('email.verify', { id: 1 });

  expect(URL.hasValidSignature(url)).toBe(true);
});

test('tampered signed route is invalid', () => {
  const url = URL.signedRoute('email.verify', { id: 1 });
  const tampered = url.replace('id=1', 'id=999');

  expect(URL.hasValidSignature(tampered)).toBe(false);
});
```

## API Reference

### `Hash`

| Method                      | Description                                                           |
| --------------------------- | --------------------------------------------------------------------- |
| `Hash.make(value, rounds?)` | Hash a plain-text value with bcrypt                                   |
| `Hash.check(plain, hashed)` | Verify a value against a stored hash                                  |
| `Hash.needsRehash(hashed)`  | Returns `true` if the hash was made with fewer rounds than configured |

### `Crypt`

| Method                           | Description                                                |
| -------------------------------- | ---------------------------------------------------------- |
| `Crypt.encryptString(value)`     | Encrypt a plain-text string; returns a base64 envelope     |
| `Crypt.decryptString(encrypted)` | Decrypt a ciphertext; throws `DecryptException` on failure |
| `Crypt.fake()`                   | Replace with a no-op identity codec for tests              |

### `URL`

| Method                                         | Description                                         |
| ---------------------------------------------- | --------------------------------------------------- |
| `URL.signedRoute(name, params?)`               | Generate a permanently signed URL                   |
| `URL.temporarySignedRoute(name, ttl, params?)` | Generate a signed URL that expires in `ttl` seconds |
| `URL.hasValidSignature(url)`                   | Verify a signature without middleware               |
